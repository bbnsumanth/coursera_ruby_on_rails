Lec 4

  • $rails new app_name => this will create a new rails app with the name app_name
  • $cd app_name $rails server(or rails s) => this will start the rails app on localhost:3000,rails comes with inbuilt server to be used in development mode
  • all the static files goes into public directory in rails app.these are directly served by rails server without using any controllers.we can access the files in public directory by directly giving the file name(localhost:3000/filename)

Lec 5

  • $ rails generate controller controllerName action1 action2 => this will generate a new controller class with the name controllerName_controller.rb and all the other related stuff like routes for the actions in controller ,views for the actions with name actionName.html.erb , javascript(coffeescript),css(scss) tests etc. Instead of generate we can use a shortcut g.
  • notice .erb extension to files in view.Erb is a templating library which lets us write ruby in .html file.this allows us to embed dynamic content into our static html code.
    • tag patterns in erb
      1. <% ...ruby code... %> this tag pattern evaluate the ruby code inside it
      2. <%= ...ruby code... %> this will return the evaluated result from ruby code
    • eg:
      • <% random_names = ["Alex","Joe","Micheal"]%>
      • Greeting, <%= random_names.sample %>

Lec 6

  • if there are no static files for a request rails will look in routes for a controller which is matching the request
  • following is the lifecycle from Request => Response
    • routes forward the request to controller =>
    • controller then talks to models,gets data etc =>
    • controller then fills the view wuith data =>
    • return the view as a response
  • rake is the build system of ruby,it is written in ruby,rake --tasks shows all the possible commands of rake,rake --describe taskname will describe the rake task

Lec 8 - Helpers

  • Helpers contain code that is used to format views
  • code in a helper for a controller can be accessed from any view that belongs to any controller
  • rails comes with many build in helpers like link_to

Lec 9 - gems(httparty)

  • RubyGems is a package manger for ruby
  • httparty is a gem to consume restful web services
  • Restfull services are simple webservices implemented using http:
    • have a base uri
    • data exchange is using xml / json
    • support http methods like get,post etc
  • httparty makes our app as a client to some web service
  • it automaticallyparse json or xml into ruby data structures.
  • params are specified in request using ? .Two params are seperated by &

Lec 10 - Bundler

  • bundler lets specify gems(and associated gem dependencies) for rails app in side Gemfile
  • It is the preferred way to manage dependencies in rails app
  • running bundle install or bundle will fetch all the dependencies mentioned in Gemfile
  • running bunble update will update the version of a gem
  • eg:
    1. gem "nokogiri"
    2. gem "nokogiri" , "3.0.0.beta3"
    3. gem "rack" ,">=1.0"
    4. gem "thin", ">=1.1 , "< 2.0"
    5. gem "thin" , "~>1.1"
  • Gemfile.lock is the file which contain exact versions of our gems

Lec 12 - Styling views,query params

  • we can define styles in the assets/controller.scss file for a controller,these are automatically applied to all the views for that controller
  • in the actions in a controller we can access the params using params[:param_name]
  • we can make a controller to serve on root path by using root 'controller#action' in routes.rb file
